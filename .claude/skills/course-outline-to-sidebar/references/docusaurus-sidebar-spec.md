# Docusaurus Sidebar Specification

Complete reference for Docusaurus sidebar configuration.

## Sidebar Types

### Document Link

Simple reference to a single document:

```typescript
'doc-id'  // Short form
// or
{
  type: 'doc',
  id: 'doc-id',
  label: 'Custom Label',
}
```

### Category

Group of documents or other categories:

```typescript
{
  type: 'category',
  label: 'Category Label',
  items: ['doc-1', 'doc-2'],
  // Optional fields:
  collapsible: true,  // Default: true
  collapsed: false,   // Default: true
  link: {
    type: 'generated-index',
    description: 'Category description',
    slug: '/category/custom-slug',
  }
}
```

### Generated Index

Auto-generated index page for a category:

```typescript
{
  type: 'category',
  label: 'Module 1',
  link: {
    type: 'generated-index',
    title: 'Module 1 Overview',
    description: 'Introduction to module 1 topics.',
    slug: '/module-1',
    keywords: ['keyword1', 'keyword2'],
  },
  items: [/* ... */],
}
```

### Link

External or internal link:

```typescript
{
  type: 'link',
  label: 'External Resource',
  href: 'https://example.com',
}
```

## Document ID Format

Document IDs are relative paths from the `docs/` directory without the `.mdx` extension:

```
docs/
├── index.mdx                    → 'index'
├── intro.mdx                    → 'intro'
├── module-1/
│   └── week-1/
│       └── chapter.mdx          → 'module-1/week-1/chapter'
```

## Best Practices

### Naming Conventions

1. **Kebab-case for files and folders**: `my-chapter-title.mdx`
2. **Prefix with numbers for ordering**: `1-intro.mdx`, `2-basics.mdx`
3. **Descriptive slugs**: Reflect content, not just generic names

### Structure Organization

```typescript
const sidebars = {
  tutorialSidebar: [
    'index',
    'introduction',
    {
      type: 'category',
      label: 'Module 1: Topic',
      link: {
        type: 'generated-index',
        description: 'Overview of module 1.',
      },
      items: [
        {
          type: 'category',
          label: 'Week 1-2: Subtopic',
          items: [
            'module-1/week-1-2/1-chapter',
            'module-1/week-1-2/2-chapter',
          ],
        },
      ],
    },
  ],
};
```

### Auto-generated Sidebars

Docusaurus can auto-generate sidebars from directory structure:

```typescript
{
  type: 'autogenerated',
  dirName: 'module-1',
}
```

Requires `_category_.json` files in directories:

```json
{
  "label": "Module 1",
  "position": 1,
  "link": {
    "type": "generated-index"
  }
}
```

## Sidebar Item Properties

### Common Properties

- `type`: Item type (`'doc'`, `'category'`, `'link'`)
- `label`: Display text (overrides frontmatter title)
- `className`: Custom CSS class

### Category-specific

- `items`: Array of child items
- `collapsible`: Whether category can collapse (default: `true`)
- `collapsed`: Initial collapsed state (default: `true`)
- `link`: Category link configuration

### Doc-specific

- `id`: Document identifier
- `customProps`: Custom metadata

## Migration from v2 to v3

Key changes in Docusaurus v3:

1. TypeScript config preferred over JavaScript
2. `type: 'doc'` can be shortened to just the doc ID string
3. Better support for generated indexes
4. Enhanced type safety

## Common Patterns

### Multi-level Nesting

```typescript
{
  type: 'category',
  label: 'Level 1',
  items: [
    {
      type: 'category',
      label: 'Level 2',
      items: [
        'level-1/level-2/doc',
      ],
    },
  ],
}
```

### Mixed Content

```typescript
{
  type: 'category',
  label: 'Resources',
  items: [
    'resources/guide',
    {
      type: 'link',
      label: 'API Docs',
      href: 'https://api.example.com',
    },
    'resources/faq',
  ],
}
```

## Troubleshooting

### Common Issues

1. **Broken links**: Ensure doc IDs match file paths exactly
2. **Missing documents**: Check that MDX files exist and have frontmatter
3. **Duplicate IDs**: Each document ID must be unique
4. **Category not showing**: Verify `items` array is not empty

### Debugging

Enable verbose logging:

```bash
npm run start -- --verbose
```

Check build output for sidebar warnings.
